# Revenue tracking

Revenue tracking allows you to monitor subscription revenue, analyze trends, and measure the financial impact of your monetization strategies.

## What is revenue tracking?

Revenue tracking captures and reports all subscription-related financial events, including:

- New subscriptions
- Renewals
- Upgrades and downgrades
- Refunds
- Trial conversions

## Automatic revenue tracking

Helium automatically tracks revenue events:

### Purchase events

- **Initial purchase**: First subscription purchase
- **Renewal**: Automatic subscription renewal
- **Upgrade**: User upgrades to higher tier
- **Downgrade**: User downgrades to lower tier

### Revenue events

- **Trial started**: Free trial begins (no revenue)
- **Trial converted**: Trial converts to paid subscription
- **Refund**: Subscription refunded
- **Cancellation**: Subscription cancelled (future revenue impact)

## Revenue metrics

### Key metrics tracked

- **Total revenue**: All subscription revenue
- **Monthly recurring revenue (MRR)**: Normalized monthly revenue
- **Annual recurring revenue (ARR)**: Normalized annual revenue
- **Average revenue per user (ARPU)**: Revenue divided by users
- **Lifetime value (LTV)**: Predicted total revenue per user

### Cohort analysis

Track revenue by user cohorts:
- Install date cohorts
- Subscription start cohorts
- Geographic cohorts
- Platform cohorts

## Revenue attribution

### Source attribution

Track revenue by acquisition source:

```swift
// iOS example - Set attribution
Helium.shared.setUserProperty("acquisition_source", value: "facebook_ads")
```

Analyze which sources drive highest revenue.

### Campaign attribution

Attribute revenue to specific campaigns:

```swift
// iOS example
Helium.shared.setUserProperty("campaign", value: "summer_sale_2024")
```

### Experiment attribution

Revenue is automatically attributed to experiments:
- Control vs variant revenue
- Statistical significance
- Revenue lift calculations

## Revenue events in code

While Helium tracks revenue automatically, you can access revenue events:

```swift
// iOS example - Listen for revenue events
Helium.shared.onRevenueEvent { event in
    print("Revenue event: \(event.type)")
    print("Amount: \(event.amount)")
    print("Product: \(event.productId)")
}
```

## Revenue validation

### Receipt validation

Helium automatically validates receipts:

- **iOS**: App Store receipt validation
- **Android**: Google Play purchase verification

This ensures revenue data is accurate and prevents fraud.

### Server-to-server notifications

Helium processes server notifications from app stores:

- Real-time revenue updates
- Renewal notifications
- Cancellation alerts
- Refund notifications

## Revenue reconciliation

### App store reports

Compare Helium revenue data with app store reports:

1. Export revenue from Helium dashboard
2. Download app store financial reports
3. Reconcile differences
4. Investigate discrepancies

### Common discrepancies

- **Timing differences**: Events may be reported at different times
- **Refunds**: May appear in different periods
- **Currency conversion**: Exchange rate differences
- **Tax handling**: Gross vs net revenue

## Troubleshooting

### Missing revenue events

If revenue isn't tracking:

1. Verify SDK integration
2. Check receipt validation
3. Confirm server notifications are enabled
4. Review error logs

### Incorrect amounts

If revenue amounts are wrong:

1. Check currency conversion settings
2. Verify tax handling configuration
3. Confirm product pricing in app stores
4. Review refund processing

### Delayed reporting

Revenue may be delayed due to:
- App store processing time
- Server notification delays
- Time zone differences
- Batch processing schedules
