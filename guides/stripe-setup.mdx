## Context

- Stripe **one-click** is a way to invoke the **Apple Pay** flow directly, rather than kicking out to a web view with a stripe portal, and keep track of entitlements all within **Helium**. It works with both native iOS flows and Revenue Cat + iOS.
- As of March 2026, we estimate that 79-81% of customers on modern versions of iOS have Apple Pay enabled. In the US, the percentage is higher: close to 82-83%.
- Currently, all payment processing is handled directly through Stripe so you can trust their infrastructure and security.

## Prerequisites

<AccordionGroup>
  <Accordion title="Create a Stripe organization on the Stripe Dashboard">
    Go to the [Stripe Dashboard ](http://dashboard.stripe.com/)and create a new business account.

    - You will need your Tax ID (EIN)
    - Bank account
    - Identity for verification
  </Accordion>
  <Accordion title="Add your Products to your Stripe organization">
    Generally this is done through the Stripe dashboard UI.
  </Accordion>
  <Accordion title="Import Your Stripe Products into Helium">
    <Warning>
      _Helium currently only ingests products with recurring prices_ 
    </Warning>
    - Helium generally expects separate products for different prices
      - We do this to enable trial product creation through the Helium dashboard
    - Caveat, you can create a product with a one-time price, as long as you add a recurring price.
      - As mentioned above, only recurring prices will be ingested into your Helium products
    - To add a trial once your products have been ingested
      - Navigate to the integrations page
      - Filter your buckets by Stripe buckets
      - Click `View Details` on the relevant bucket, and tab to offers, and add a trial
      - Trials are currently set in number of days
      - Trial entitlement is set at a product level in Helium
    - It is generally best practice to sync your products often, and after any changes made on the Stripe dashboard
  </Accordion>
  <Accordion title="Register your Apple Merchant ID">
    Better documentation on this step can be found here: [Apple](https://developer.apple.com/help/account/capabilities/configure-apple-pay#create-a-merchant-identifier)
  </Accordion>
  <Accordion title="Authenticate Stripe with Certificates">
    Go to the [iOS Certificate Settings](https://dashboard.stripe.com/settings/ios_certificates) in the Stripe Dashboard, click **Add new application**, and follow the guide.

    Download a Certificate Signing Request (CSR) file to get a secure certificate from Apple that allows you to use Apple Pay.

    One CSR file must be used to issue exactly one certificate. If you switch your Apple Merchant ID, you must go to the [iOS Certificate Settings](https://dashboard.stripe.com/settings/ios_certificates) in the Dashboard to obtain a new CSR and certificate.
  </Accordion>
</AccordionGroup>

## App Integration

<Warning>
  In order to show paywalls with Stripe or Paddle products, you are **required** to set a custom user ID. This ID will be used to grant entitlements across devices and re-installs.
</Warning>

<Tabs>
  <Tab title="iOS">
    <Note>
      Stripe integration requires Helium **4.1.8+**. If you have not integrated Helium yet, follow the [iOS SDK Quickstart](/sdk/quickstart-ios) first.
    </Note>
    <Steps>
      <Step title="Add Apple Pay Capability to Xcode">
        In Xcode, open your project settings, click the **Signing & Capabilities** tab, and add the **Apple Pay** capability. Select the merchant ID you created earlier.
      </Step>
      <Step title="Add StripeOneTapPurchase Dependency">
        Add this Swift Package dependency:

        ```text
        https://github.com/appmonetization/stripe-one-tap-purchase.git
        ```
      </Step>
      <Step title="Modify your Helium Initialize Call">
        Replace `Helium.shared.initialize` with `Helium.shared.initializeWithStripeOneTap`:

        ```swift
        Helium.shared.initializeWithStripeOneTap(
            apiKey: "your-helium-api-key",
            stripePublishableKey: "pk_live_...",
            backupPurchaseDelegate: StoreKitDelegate(), // or RevenueCatDelegate
            merchantIdentifier: "merchant.com.example",
            merchantName: "Your App Name",
            managementURL: URL(string: "https://your-app.com/manage-subscription")! // e.g. link to generic Stripe portal or your website
            // countryCode: "US"   // optional, defaults to "US"
            // currencyCode: "USD"  // optional, defaults to "USD"
        )
        ```

        The `backupPurchaseDelegate` will be used for any non-Stripe products.
      </Step>
    </Steps>
    <Accordion title="Additional Stripe Functions">
      These are available on `Helium.shared` after initializing with Stripe:

      - **`setUserIdAndSyncStripeIfNeeded(userId:)`** — Sets the user ID and syncs Stripe entitlements if needed. Call this instead of setting `Helium.identify.userId` if Helium is already initialized.
      - **`resetStripeEntitlements(clearUserId:)`** — Resets Stripe entitlements and optionally clears the user ID. Use this to "log out" a Stripe user if your app supports multiple Stripe users on the same device.
      - **`createStripePortalSession(returnUrl:)`** — Creates a Stripe Customer Portal session and returns the portal URL. Open this in a browser or in-app webview to let the user manage their subscriptions.

        ```swift
        let portalURL = try await Helium.shared.createStripePortalSession(
            returnUrl: "myapp://settings"
        )
        await UIApplication.shared.open(portalURL)
        ```
      - **`hasActiveStripeEntitlement()`** — Returns `true` if the user has any active Stripe entitlement (e.g. an active subscription).

        ```swift
        let hasEntitlement = await Helium.shared.hasActiveStripeEntitlement()
        ```
    </Accordion>
    <Accordion title="External Web Checkout (App-to-Web)">
      If you're using external web checkout (instead of Apple Pay one-tap), configure redirect URLs so Helium can detect when checkout completes or is cancelled. This flow supports Stripe and Paddle products.

      <Warning>
        **Required for external web checkout.** If you don't call `enableExternalWebCheckout`, paywalls with Stripe or Paddle products show a fallback view instead of the checkout flow.
      </Warning>

      **1. Register a URL scheme for your app**

      Add a custom URL scheme to your `Info.plist` so your app can receive checkout redirects:

      ```xml
      <key>CFBundleURLTypes</key>
      <array>
        <dict>
          <key>CFBundleURLSchemes</key>
          <array>
            <string>myapp</string>
          </array>
        </dict>
      </array>
      ```

      **2. Enable external web checkout with your redirect URLs**

      Call `enableExternalWebCheckout` after Helium initializes:

      ```swift
      Helium.config.enableExternalWebCheckout(
          successURL: "myapp://checkout-success",
          cancelURL: "myapp://checkout-cancelled"
      )
      ```

      Both URLs must use a scheme your app has registered. When the user completes or cancels checkout, Helium detects the redirect to close the checkout view and fulfill entitlements.

      **3. (Optional) Configure checkout presentation style**

      Control how the Stripe checkout page displays:

      ```swift
      Helium.config.stripeCheckoutStyle = .safariInApp
      ```

      Options:
      - `.externalBrowser` — Opens checkout in the system's default browser (default)
      - `.safariInApp` — Presents checkout using SFSafariViewController
      - `.webView` — Presents checkout in an embedded WKWebView within your app

      **4. (Optional) Disable external web checkout**

      Disable external web checkout and revert to fallback behavior:

      ```swift
      Helium.config.disableExternalWebCheckout()
      ```

      <Note>
        If the app is terminated during a checkout session, Helium automatically recovers and completes any pending checkouts when the app next initializes.
      </Note>

      **Targeting with `stripeCheckoutEligible`**

      The SDK automatically sets a `stripeCheckoutEligible` targeting parameter that you can use in your workflow audience rules. This parameter is `true` when all of the following conditions are met:

      - External web checkout is enabled (you've called `enableExternalWebCheckout`)
      - The device has compatible payment cards (Visa, Mastercard, Amex, or Discover)
      - The device is running iOS 16.4 or later

      Use this parameter to target users who can complete the external web checkout flow. See [Targeting your Stripe Paywall](#workflows-should-target-us-storefront-and-apple-pay-enabled-only) for setup instructions.
    </Accordion>
    <Accordion title="Paddle Functions">
      If you're using Paddle products via [external web checkout](#external-web-checkout-app-to-web), the following functions are available on `Helium.shared`:

      - **`resetPaddleEntitlements(clearUserId:)`** — Resets Paddle entitlements and optionally clears the user ID. Use this to "log out" a Paddle user if your app supports multiple Paddle users on the same device.

        ```swift
        Helium.shared.resetPaddleEntitlements(clearUserId: true)
        ```

      - **`hasActivePaddleEntitlement()`** — Returns `true` if the user has any active Paddle entitlement (e.g. an active subscription).

        ```swift
        let hasPaddleEntitlement = await Helium.shared.hasActivePaddleEntitlement()
        ```
    </Accordion>

## Checking Entitlements

The Helium SDK provides entitlement helper methods that work with Stripe and Paddle purchases. See the entitlements section in the [iOS Quickstart](/sdk/quickstart-ios#checking-subscription-status--entitlements), [React Native Quickstart](/sdk/quickstart-react-native#checking-subscription-status--entitlements), or [Flutter Quickstart](/sdk/quickstart-flutter#checking-subscription-status--entitlements) for the full API reference.

<Warning>
  On iOS, some entitlement methods (e.g. `hasActiveSubscriptionFor(subscriptionGroupID:)`, `subscriptionStatusFor(...)`, `activeSubscriptions()`) rely on StoreKit and will **not** reflect Stripe or Paddle purchases. For Stripe and Paddle, use `hasEntitlementForPaywall(trigger:)`, `hasActiveEntitlementFor(productId:)`, `hasAnyActiveSubscription()`, `hasAny()`, `hasActiveStripeEntitlement()`, and `hasActivePaddleEntitlement()` instead.
</Warning>

## Creating and Target your Stripe Paywall

<Steps>
  <Step title="In your Helium Dashboard, create a paywall and add your Stripe product buckets">
    <Danger>
      At the moment, each paywall can include **iOS-only** buckets and **Stripe-only** buckets on the same paywall. However, you cannot mix **Stripe-only** buckets with **iOS+Android** buckets on the same paywall.
    </Danger>
    Once you've done the step above to import your Stripe products to Helium, you can create a paywall and adding your product bucket with the Stripe product. This product will behave the same way as all Helium product buckets (except for the caveat above).
  </Step>
  <Step title="Add Stripe Products and iOS Products as Alternative">
    When creating a paywall you will add your product buckets, then you will need to direct the editor to make the appropriate visual changes to show the Stripe products at once and the iOS products in another view. 

    For example, if you had a `pro_plus` as an iOS product and `pro_plus` as a Stripe product you would:

    1. Add both products to the paywall
    2. Prompt the editor to present the Stripe product in the primary view, and have a "Show Other Payment Options" button that would bring up the secondary view with the iOS product.
    3. Test the functionality right in the editor preview using the action log.

    <Tip>
      It's generally best practice to pair each Stripe product with a corresponding iOS product. Present only Stripe products by default, surfacing iOS products only when absolutely necessary.
    </Tip>
  </Step>
  <Step title="Workflows should target US storefront and Apple Pay enabled only">
    <Warning>
      This is required to make sure that paywalls with Stripe are shown in the US.
    </Warning>
    When adding paywalls to a workflow make sure to use a Stripe eligible targeting rule that selects:

    - Country Storefront: US
    - Apple Pay: Enabled

    To make this easy we've added a button within audeince creation to easily add the neccessary attributes for Stripe targeting. Navigate to Targeting \> Create Audience \> Press the 3 dot menu in **Targeting Rules** \> add Stripe Eligible!

    <Frame>
      ![Screenshot 2026 03 11 At 1 17 11 PM](/images/Screenshot2026-03-11at1.17.11PM.png)
    </Frame>

    **For External Web Checkout (iOS)**

    If you're using the [external web checkout flow](#external-web-checkout-app-to-web) instead of Apple Pay one-tap, you can use the `stripeCheckoutEligible` targeting parameter for more precise targeting. This parameter checks that the user:
    - Has external web checkout enabled in your app configuration
    - Has compatible payment cards on their device
    - Is running iOS 16.4 or later (required for the Apple Pay web flow)

    Add `stripeCheckoutEligible: true` to your audience targeting rules to show external web checkout paywalls only to eligible users.
  </Step>
</Steps>

## FAQ

- Does this work on Android?
  - Not yet, we are adding support for Google Wallet payments soon.
- How do entitlements work?
  - Helium provides some methods to retrieve current entitlements of Stripe customers.
- How do trial offers work?
  - Once your product has been imported from Stripe, you must use the Helium dashboard to actually add a trial product.
  - Due to limitations of the Stripe dashboard.
- Can I do one-time purchases / consumables?
  - Not yet, coming soon.
- Can retention and re-subscribe flows move from iOS/App Store to this Stripe flow?
  - Yes, users who have made a purchase or have an existing entitlement to App Store can still make Stripe purchases. Note however that any existing App Store subscriptions will not automatically be canceled.
- Do you support RevenueCat?
  - Yes, but this requires some extra configuration so that RevenueCat can include Stripe products in their entitlements logic.
- Do I need to present a paywall to use this Stripe flow?
  - At the moment, yes.
- How do I handle subscription management?
  - Cancels and subscription management is handled by Stripe. The Helium SDK provides a method to access the Stripe portal.
  - On iOS the SDK method is `Helum.shared.createStripePortalSession(returnUrl:)`
  - On React Native the SDK method is `await createStripePortalSession(returnUrl)`
