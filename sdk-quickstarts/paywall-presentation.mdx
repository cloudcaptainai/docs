# Paywall presentation notes

Learn how to effectively present paywalls in your app to maximize conversions while maintaining a great user experience.

## When to show paywalls

Consider these common trigger points:

- **App launch**: Show to new users or after trial expiration
- **Feature access**: Gate premium features behind subscriptions
- **Usage limits**: Present when users hit free tier limits
- **Onboarding completion**: Show after users understand your app's value
- **Custom events**: Trigger based on specific user actions

## Presentation styles

### Modal presentation

Full-screen or sheet-style paywalls that require user interaction to dismiss.

```swift
// iOS example
Helium.shared.presentPaywall(style: .modal)
```

### Inline presentation

Embed paywalls within your existing UI flow.

```swift
// iOS example
let paywallView = Helium.shared.getPaywallView()
containerView.addSubview(paywallView)
```

## User experience considerations

### Don't interrupt critical flows

Avoid showing paywalls during:
- Active user tasks
- Form completion
- Content consumption

### Provide clear value

Before presenting a paywall:
- Demonstrate your app's value
- Explain what users get with a subscription
- Show social proof or testimonials

### Allow dismissal

Always provide a way to dismiss paywalls (except for hard paywalls where appropriate).

## Testing presentation

Use StoreKit testing (iOS) or test environments (Android) to verify:
- Paywall loads correctly
- Products display properly
- Purchase flow works end-to-end
- Dismissal behavior is correct

## Fallback handling

Always implement fallback paywalls for offline scenarios. See the fallback documentation for details.
