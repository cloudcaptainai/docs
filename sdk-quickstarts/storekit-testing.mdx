# StoreKit testing

StoreKit testing allows you to test in-app purchases and subscriptions on iOS without processing real transactions or using sandbox accounts.

## StoreKit configuration files

StoreKit configuration files define your products locally for testing.

### Create a configuration file

1. In Xcode, go to **File > New > File**
2. Select **StoreKit Configuration File**
3. Name it (e.g., `Products.storekit`)
4. Add your subscription products

### Configure products

Define products matching your App Store Connect setup:

```json
{
  "identifier": "premium_monthly",
  "type": "auto-renewable-subscription",
  "displayName": "Premium Monthly",
  "description": "Monthly premium subscription",
  "price": "9.99",
  "familyShareable": false,
  "subscriptionGroupIdentifier": "premium_group"
}
```

## Enable StoreKit testing

### In Xcode

1. Select your scheme
2. Edit Scheme > Run
3. Options tab
4. Set StoreKit Configuration to your `.storekit` file

### In code

```swift
// No code changes needed - Helium SDK automatically works with StoreKit testing
Helium.shared.presentPaywall()
```

## Testing scenarios

### Purchase flows

Test the complete purchase experience:
- Product display
- Purchase initiation
- Transaction processing
- Subscription activation

### Subscription states

Simulate different subscription states:
- **Active**: User has valid subscription
- **Expired**: Subscription lapsed
- **In grace period**: Payment failed but access continues
- **In billing retry**: Attempting to resolve payment issue

### Renewals and cancellations

StoreKit testing accelerates subscription timelines:
- Monthly subscriptions renew every 5 minutes
- Annual subscriptions renew every 1 hour
- Test cancellations and resubscriptions

## StoreKit testing vs sandbox

| Feature | StoreKit Testing | Sandbox |
|---------|------------------|---------|
| Setup | Local configuration file | App Store Connect + test account |
| Speed | Instant | Requires network |
| Offline | ✅ Works offline | ❌ Requires connection |
| Real receipts | ❌ Simulated | ✅ Real sandbox receipts |
| Best for | Development | Pre-production testing |

## Debugging

### Transaction manager

Use Xcode's Transaction Manager to:
- View all transactions
- Clear purchase history
- Simulate renewal failures
- Test refunds

Access via: **Debug > StoreKit > Manage Transactions**

### Console logs

Enable verbose logging:

```swift
Helium.shared.setLogLevel(.verbose)
```

## Limitations

StoreKit testing doesn't simulate:
- App Store receipt validation
- Server-to-server notifications
- Promotional offers (some scenarios)
- Family Sharing

For these scenarios, use sandbox testing.

## Best practices

1. **Test early**: Use StoreKit testing during development
2. **Sandbox before release**: Always test with sandbox before production
3. **Clear state**: Reset transactions between test runs
4. **Match production**: Keep StoreKit config in sync with App Store Connect
5. **Test edge cases**: Simulate failures, cancellations, and upgrades
